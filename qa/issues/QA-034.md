# QA-034 — Resume button is never disabled while a scan is running (_set_running omits it)

- **group**: R3   **disposition**: FIX   **severity**: low   **dimension**: ui-ux
- **dedup_key**: `resume-button-not-greyed-while-running`
- **location**: scanner/scanops_scanner_gui.py (_set_running toggles only preview_btn/start_btn/stop_btn; the 재개 실행 button has no stored reference)

## Evidence
The resume button is created inline with no `self.*` reference and is never passed to `_set_running`, so it stays "normal" throughout a running scan while every other action button greys out. Behaviorally safe (the `_start` guard `if self.proc is not None: showwarning` prevents a second process), but it is an inconsistent affordance and lets the user repeatedly trigger the modal during a long scan.

## Proposed fix
Store `self.resume_btn` and include it in `_set_running` so it disables while `self.proc is not None`, matching preview/start; keep the `_start` guard as backstop.

## Notes
Not covered by QA-023/025/009. Low severity (cosmetic, no functional bug) but cheap and consistent. Verified in code.
