# QA-011 — phase1 'precision' single-run preset reintroduces the fatal -sU + --version-all combo the auto workflow was redesigned to avoid

- **group**: F    **disposition**: FIX    **severity**: high    **dimension**: nmap-coverage
- **dedup_key**: `phase1_preset_version_all_on_udp_chatty_ports`
- **location**: scanner/scanops_scanner.py:83-94 (PRESETS['phase1']), :26-27 (UDP_DEFAULT_PORTS / PRECISION_PORTS); exposed via GUI scanner/scanops_scanner_gui.py:29 ('single_precision' -> 'phase1'), :36 description '전체 포트 정밀 확인'

## Evidence
```
phase1 = ["-sS", "-sU", "-Pn", "-n", "-sV", "--version-all", "--open", ... "-p", PRECISION_PORTS ...] with PRECISION_PORTS = "T:1-65535,U:7,53,...,161,...,1900,5353,5355,11211". This applies -sU with --version-all (intensity 9) to chatty/amplification UDP ports (53 DNS, 161 SNMP, 1900 SSDP, 5353 mDNS). The code's own comments at :70-71 state exactly this is dangerous: '--version-all ... 수다스러운/증폭형 UDP 서비스(SNMP·SSDP·DNS 등)에서 ... nmap 을 fatal 종료시킬 위험이 크고' — which is why AUTO_UDP_IDENTIFY_FLAGS (:72-76) deliberately omits --version-all.
```
## Impact
The GUI 'single - 전체 포트 정밀 확인' (precision) mode and CLI --workflow single --profile phase1 can trigger the same nmap fatal exit on noisy/amplification UDP responses that ISSUE-001's redesign was built to prevent. Users choosing the 'most thorough' option get the least reliable, crash-prone scan. The protection added to auto was never propagated to the equivalent single preset.

## Proposed fix
Remove --version-all from PRESETS['phase1'] (rely on default -sV intensity 7 for the UDP portion), or split phase1 so --version-all applies only to the TCP port range and not to -sU/U: ports — mirroring the auto stages' separation.

## Verifier rationale
Confirmed from code. PRESETS['phase1'] (scanner/scanops_scanner.py:87-93) contains both "-sU" and "--version-all" together with "-p", PRECISION_PORTS. PRECISION_PORTS (line 27) = f"T:1-65535,U:{UDP_DEFAULT_PORTS}", and UDP_DEFAULT_PORTS (line 26) includes the exact chatty/amplification UDP ports the finding names (53 DNS, 161 SNMP, 1900 SSDP, 5353/5355 mDNS/LLMNR, 11211 memcached). The module's OWN comment at lines 70-71 states --version-all (intensity 9) on chatty/amplification UDP services (SNMP/SSDP/DNS) risks fatally terminating nmap, which is precisely why AUTO_UDP_IDENTIFY_FLAGS (72-76) deliberately omits --version-all. That protection is NOT propagated to phase1. The flow is grounded: build_base_flags (line 289) copies PRESETS[profile] verbatim and never strips --version-all (only -sU is stripped, and only under --tcp-only at line 293); build_command for the single workflow (line 417, empty stage_id) emits plan["base_flags"] directly into the nmap command (lines 420-426). GUI mapping confirmed: scanops_scanner_gui.py:29 maps single_precision -> phase1, and lines 22/36 label/describe it as the most-thorough "전체 포트 정밀 확인" option. Default GUI scan type is "프로필 기본" so -sU is retained by default, meaning the -sU + --version-all combo reaches nmap unmodified. This is distinct from ISSUE-001 (which concerns fail_plan aborting the plan on UDP stage exit!=0): this finding is the fatal nmap exit itself being reintroduced in the single/phase1 code path that the auto redesign never touched. Real, in-scope reliability defect.
