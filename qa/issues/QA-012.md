# QA-012 — "No live hosts" / "no open ports" is reported as a clean rc=0 skip, not honestly surfaced as an empty result

- **group**: G    **disposition**: FIX    **severity**: high    **dimension**: result-correctness
- **dedup_key**: `no-summary-empty-result-indistinguishable-from-success`
- **location**: scanner/scanops_scanner.py:728-729, 752-754 (append_skipped_stage); 654-664 (finish_plan); 620-638 (write_manifest)

## Evidence
```
A valid discovery XML with all hosts `status state="down"` produces `live_hosts_from_xml -> []` and `open_ports_from_xml -> []` (verified), identical to the corrupt-XML case. finish_plan prints only `done: <manifest>` and exits 0; manifest import_xml_files is empty. There is no summary line stating how many hosts were up or how many open ports were found. The whole-plan terminal output for an empty result is byte-for-byte the same shape as a successful one.
```
## Impact
An operator cannot tell from exit code or stdout whether the scan (a) found a live network with no open ports, (b) found nothing because all targets were down/unreachable, or (c) broke (see finding #1). All three end with exit 0 and 'done'. There is no honest '0 hosts up / 0 open ports' signal, so empty/failed scans are easily mistaken for confirmed-clean networks.

## Proposed fix
In finish_plan (or after the per-batch loop), aggregate and print a summary: number of live hosts found in discovery, number of open TCP/UDP ports, number of import XML files. Consider a distinct non-zero exit (or explicit status like 'done_no_results') when zero live hosts were found across all batches, so callers/GUI can flag it.
