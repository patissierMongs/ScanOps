# QA-044 — finalize_plan status=="failed" / exit-code-1 path (all stages fail, importable=0) is completely untested

- **group**: R3   **disposition**: FIX (test)   **severity**: high   **dimension**: coverage-gaps
- **dedup_key**: `failed-status-exit1-path-untested`
- **location**: scanner/scanops_scanner.py finalize_plan (`status = "failed" if failed else "done"`, `return 0 if status in ("done","partial") else 1`), print_scan_summary ("모든 단계 실패"); test harness backend/tests/test_standalone_scanner.py

## Evidence
The three exit paths of finalize_plan are partial(0), done(0), failed(1). Every existing test fails only ONE stage, so importable_xml() is always non-empty and the code lands on 'partial'. Grep of the test file shows zero assertions for `status=="failed"`, zero for a non-zero scan-engine returncode, and the fake nmap matches `FAKE_NMAP_FAIL_STAGE` against exactly one stage (no multi-stage failure). The sole non-zero exit from a completed scan plus its user-facing error message are unguarded.

## Proposed fix
Extend the fake nmap to support failing every stage with no XML (e.g. `FAKE_NMAP_FAIL_ALL`), then add a test asserting `returncode==1`, `state['status']=='failed'`, `"모든 단계 실패"` in stderr, and `manifest['import_xml_files']==[]`.
