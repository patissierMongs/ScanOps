# QA-002 — tcp_identify rc!=0 aborts the whole plan, discarding successful tcp_discovery AND skipping udp_identify (sibling of ISSUE-001)

- **group**: A    **disposition**: FIX    **severity**: high    **dimension**: failure-isolation
- **dedup_key**: `stage_rc_nonzero_fail_plan_aborts_whole_plan_tcp_identify`
- **location**: scanner/scanops_scanner.py:724-727 (execute_auto), fail_plan at :692

## Evidence
```
if not stage_succeeded(plan, idx, "tcp_identify"):
    rc = run_nmap_stage(plan, idx, state_path, "tcp_identify", tcp_ports, live_hosts)
    if rc != 0:
        return fail_plan(plan, state_path, rc)  # returns OUT of execute_auto
```
## Impact
When tcp_identify exits non-zero (very plausible: AUTO_TCP_IDENTIFY_FLAGS uses --version-all/intensity-9 which the code's own comment at :62-67 warns can make nmap die on chatty services, plus --script with --script-timeout 10s where a single NSE crash can yield a non-zero rc), the entire plan is marked 'failed' and execute_auto RETURNS. The completed tcp_discovery results are excluded from import (manifest filters returncode==0 only), and udp_identify for this batch is NEVER attempted. A single noisy TCP service thus wipes out both the open-port list and all UDP exposure data. This is the same class as ISSUE-001 but for the TCP-identify stage and it additionally prevents the later UDP stage from running at all.

## Proposed fix
Treat each stage failure as a per-stage/per-batch error: record the failed stage, continue to udp_identify and to remaining batches, and only mark the overall plan 'partial'/'completed_with_errors' at the end. Do not 'return fail_plan' from inside the stage; accumulate failures and let finish_plan run so good XML is still manifested.

## Verifier rationale
Confirmed in scanner/scanops_scanner.py. At lines 724-727 of execute_auto, when tcp_identify returns rc!=0 the code does `return fail_plan(plan, state_path, rc)`. fail_plan (lines 692-697) sets status="failed", writes the manifest, and returns WITHOUT calling finish_plan — and because it returns out of execute_auto, it exits the per-batch `for idx` loop at line 712.
