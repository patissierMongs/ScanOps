# QA-057 — Auto-workflow UDP skip for a TCP-only --ports spec (no --tcp-only flag) is untested

- **group**: R5   **disposition**: FIX (test)   **severity**: low   **dimension**: coverage-gaps
- **dedup_key**: `auto-tcp-only-ports-override-udp-skip-untested`
- **location**: scanner/scanops_scanner.py execute_auto (`elif not auto_udp_ports(plan):` skip) and auto_udp_ports (TCP-only override → "")

## Evidence
In the auto workflow, passing a TCP-only `--ports` (e.g. `22,443` or `T:80`) WITHOUT `--tcp-only` leaves tcp_only=False but makes `auto_udp_ports(plan)` return `""`, so `execute_auto` reaches the `elif not auto_udp_ports(plan)` branch and appends a skipped udp_identify with reason '사용자가 지정한 포트에 UDP 포트가 없습니다'. No test exercises this: the existing tests use the `--tcp-only` FLAG (different branch), `U:53` (discovery-skipped path), or `all_ports=True` (the opposite, `.startswith('U:')` assertion). The empty-string contract of `auto_udp_ports` for a TCP-only override and the line's skip reason are uncovered.

## Proposed fix
Add a focused test: `--workflow auto --ports 22,443 <target>` (no --tcp-only) asserting (a) udp_identify skipped=True with 'UDP 포트가 없습니다' in skip_reason, (b) tcp_discovery/tcp_identify still ran, (c) no udp XML produced. Add unit assertions `auto_udp_ports({'ports_override':'22,443'}) == ''` and `{'ports_override':'T:80'} == ''`.

## Notes
Production code is correct; pure missing-test gap. Not covered by QA-036 (all_ports, opposite case). Verified by execution.
