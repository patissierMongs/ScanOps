# QA-021 — Default auto workflow silently drops UDP-only hosts; GUI gives no way to enable full UDP coverage despite promising 'UDP service identification'

- **group**: E    **disposition**: FIX    **severity**: medium    **dimension**: nmap-coverage
- **dedup_key**: `auto-udp-identify-skips-udp-only-hosts-and-gui-lacks-udp-all-targets`
- **location**: scanner/scanops_scanner.py:743-758 (udp_all_targets opt-in vs live_hosts-only default), :817-818 (--udp-all-targets help); GUI scanner/scanops_scanner_gui.py (no --udp-all-targets control; README/GUI promise at :201-203 and README.md:58 '주요 UDP 서비스 ... 확인')

## Evidence
```
In execute_auto the default branch (:750-756) runs UDP identify only against live_hosts derived from TCP discovery (open_ports/PE/PS/PA). A host that is silent to TCP SYN, ICMP echo, and ACK-on-80/443/3389 but serves UDP (DNS/SNMP/NTP-only host) is never in live_hosts, so UDP identify excludes it (:752-754 even skips with '살아있는 호스트를 찾지 못했습니다'). Full coverage requires plan['udp_all_targets'] (:743), set only by --udp-all-targets. grep of scanner/scanops_scanner_gui.py for udp_all_targets/--udp-all-targets returns no matches — the GUI never exposes it.
```
## Impact
The headline auto-scan feature ('주요 UDP 서비스 식별', and GUI expected-results text claiming UDP services are checked) silently misses UDP-only hosts for every GUI user, and for CLI users who don't know the buried --udp-all-targets flag. For internal exposure scanning, UDP-only assets (DNS/SNMP/SSDP/IKE) are exactly high-value findings, so the default produces false-negative coverage with no warning.

## Proposed fix
Expose --udp-all-targets as a GUI checkbox (e.g. '숨은 UDP 전용 호스트도 확인'), and/or document clearly in the preview/log that default UDP identify only covers TCP/ICMP/ACK-discovered hosts so UDP-only hosts are skipped unless the option is enabled.
