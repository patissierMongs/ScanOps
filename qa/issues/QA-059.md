# QA-059 — Dry-run preview shows a UDP stage (with the invalid -sT -sU combo) for --scan-type connect that execute_auto actually skips

- **group**: R7   **disposition**: FIX   **severity**: low   **dimension**: scan-procedure / ui-ux
- **dedup_key**: `dryrun-preview-shows-udp-stage-for-connect-that-is-skipped`
- **location**: scanner/scanops_scanner.py print_plan (UDP preview gated only on tcp_only) vs execute_auto (skips udp_identify for scan_type=='connect', QA-010)

## Evidence
`print_plan` gates the UDP preview on `if not plan.get("tcp_only") and auto_udp_ports(plan):` — it does NOT check `scan_type=='connect'`. But `execute_auto` skips the UDP stage for connect. Verified: `--workflow auto --scan-type connect --dry-run 10.0.0.5` prints `# 주요 UDP 서비스 식별` followed by `... -sT -sU -Pn -n -sV --open ... -p U:...` — a UDP command that (a) will never run, and (b) is exactly the `-sT`(connect)+`-sU`(raw-socket) combination connect mode exists to avoid (QA-010). In the auto path `-sU` is never stripped for connect (set_scan_type only prepends -sT), so the only thing preventing this bad command at runtime is the execute-side skip the preview fails to mirror. `test_connect_scan_strips_udp_everywhere` asserts only the EXECUTE state; no test checks the dry-run preview.

## Proposed fix
Mirror execute_auto's skip in print_plan: `if not plan.get("tcp_only") and plan.get("scan_type") != "connect" and auto_udp_ports(plan):`, with an else/elif note that the UDP stage is skipped in connect mode (mirroring the tcp_only message style). Add a dry-run assertion that the UDP/-sU command is absent from connect preview output.

## Notes
Preview/execute divergence — the dry-run feature exists to show what will run; advertising a never-run UDP stage with the precise invalid combo QA-010 eliminated is misleading. Low severity (preview only). Verified live + end-to-end. Not covered by QA-010 (execute path only).
