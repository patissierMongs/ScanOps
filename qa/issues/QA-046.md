# QA-046 — --open-only adding --open, and the --open-only / --include-closed precedence, are untested (CLI build_base_flags + apply_auto_modifiers)

- **group**: R3   **disposition**: FIX (test)   **severity**: medium   **dimension**: coverage-gaps
- **dedup_key**: `open-only-include-closed-precedence-untested`
- **location**: scanner/scanops_scanner.py build_base_flags (`if include_closed: strip --open` then `if open_only and "--open" not in flags: append`) and apply_auto_modifiers (same logic)

## Evidence
The only test touching this region (test_phase1_can_drop_udp_scripts_and_open_filter) exercises only the include_closed REMOVE path. There is no test for the open_only ADD path on a profile lacking --open (e.g. basic), nor for the interaction with BOTH set. Verified: `build_base_flags(open_only=True)` on basic → `[...,'--open']`; `build_base_flags(open_only=True, include_closed=True)` → `[...,'--open']` (open_only wins). All three outcomes unasserted.

## Proposed fix
Add tests: `build_base_flags(open_only=True)` on basic asserts `--open` appended; both flags set asserts `--open` present (open_only precedence); and the auto analogue via apply_auto_modifiers.
