# QA-036 — --all-ports combined with a TCP-only --ports value silently disables the UDP identify stage

- **group**: R3   **disposition**: FIX   **severity**: medium   **dimension**: scan-procedure
- **dedup_key**: `all-ports-with-tcp-ports-override-kills-udp-stage`
- **location**: scanner/scanops_scanner.py:466-471 (auto_udp_ports) vs :456-463 (auto_tcp_discovery_ports); skip at execute_auto (`elif not auto_udp_ports(plan):`)

## Evidence
`auto_tcp_discovery_ports` checks `all_ports` FIRST (returns T:1-65535). `auto_udp_ports` never looks at `all_ports`: with a non-empty `ports_override` it derives UDP ports via `protocol_ports(override,'U')`, which is empty for a TCP-only override. Verified: `auto_udp_ports({'all_ports':True,'ports_override':'22,80'})==''`. Dry-run `--all-ports --ports 22,80` prints only tcp_discovery + tcp_identify; the UDP stage is appended SKIPPED. The user asked for the broadest scan yet silently loses all UDP coverage because of a leftover `--ports`, with no warning (warn_ambiguous_ports returns early for specs without ':').

## Proposed fix
Make `auto_udp_ports` honor `all_ports`: return the default UDP port set when `plan['all_ports']` is set (regardless of `ports_override`), consistent with `auto_tcp_discovery_ports`. (tcp_only is handled earlier in execute_auto.)
