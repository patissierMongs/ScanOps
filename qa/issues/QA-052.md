# QA-052 — The no-live-hosts UDP decision (default skip + --udp-all-targets rescue) is untested; the QA-021 invariant is unverified

- **group**: R4   **disposition**: FIX (test)   **severity**: medium   **dimension**: coverage-gaps
- **dedup_key**: `no-live-hosts-udp-skip-and-rescue-untested`
- **location**: scanner/scanops_scanner.py execute_auto UDP branch (default skip-with-hint when live_hosts is None; --udp-all-targets runs against the original batch); tests at backend/tests/test_standalone_scanner.py

## Evidence
When tcp_discovery returns open TCP ports but NO status=up host, `live_hosts` is None: the default workflow SKIPS udp_identify with the '...--udp-all-targets...' hint, while `--udp-all-targets` runs udp_identify against the original batch. Both existing tests use `_fake_nmap`, which always emits status='up' for 127.0.0.1, so discovery always yields a live host; neither drives `live_hosts==None`. The entire reason `--udp-all-targets` exists (and the default skip-with-hint path) is unverified — a future change could silently re-skip UDP or break the rescue and all tests still pass.

## Proposed fix
Add a fake-nmap mode whose tcp_discovery XML has an open TCP port but host status!='up' (no live hosts). Assert: (1) default run → udp_identify skipped=True with '--udp-all-targets' in skip_reason and no udp XML; (2) same input with --udp-all-targets → udp_identify runs against the original batch (FAKE_NMAP_LOG udp targets == original batch).

## Notes
Production code is correct; pure missing-test gap. Not covered by QA-021 (GUI checkbox + docs) or QA-012 (empty-result summary). Verified by live reproduction.
