# QA-031 — open_only injects --open into the auto tcp_discovery stage, breaking the UDP-only-host survival invariant

- **group**: R3   **disposition**: FIX   **severity**: high   **dimension**: ui-ux / scan-procedure
- **dedup_key**: `open-only-adds-open-to-discovery-breaks-udp-only-survival`
- **location**: scanner/scanops_scanner.py:474-486 (apply_auto_modifiers, open_only append) applied to every stage via build_auto_flags; GUI passes --open-only ungated at scanner/scanops_scanner_gui.py (_command open_only branch)

## Evidence
`apply_auto_modifiers` does `if plan.get("open_only") and "--open" not in flags: flags.append("--open")` with no stage gating, so it appends `--open` to `tcp_discovery`, whose `AUTO_TCP_DISCOVERY_FLAGS` deliberately omits it. Dry-run `--workflow auto --open-only` emits a `tcp_discovery` line containing `--open`. With `--open`, nmap drops up-hosts that have 0 open TCP ports from the XML, so `live_hosts_from_xml` returns fewer hosts and `udp_identify` (run only against discovery live hosts) silently skips UDP-only hosts (DNS/SNMP/NTP). Violates the documented invariant (scanner.py:55-57) and the gap is not caught by `test_discovery_omits_open_so_udp_only_hosts_survive` (only exercises the default path).

## Proposed fix
Make `apply_auto_modifiers` stage-aware: never add `--open` to `tcp_discovery` (pass `stage_id`, skip the open_only append for discovery). Identify stages already carry `--open`, so open-only remains effective where intended. Add a regression test asserting `--open` absent in discovery, present in identify, under `--open-only`.

## Notes
Distinct from QA-025 (toggle no-op/conflict, GUI display) and QA-021 (default-path UDP-only gap). Verified empirically; not overengineering (restores a documented, test-asserted invariant).
