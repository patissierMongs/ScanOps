# QA-038 — Auto workflow: a scan whose only successful stage is tcp_discovery is mislabeled 'failed' (exit 1) despite live hosts with open ports

- **group**: R3   **disposition**: FIX   **severity**: high   **dimension**: result-reliability
- **dedup_key**: `discovery-only-success-mislabeled-failed`
- **location**: scanner/scanops_scanner.py importable_xml (skips tcp_discovery unconditionally), finalize_plan (status=failed when importable empty + failed), print_scan_summary ("모든 단계 실패"), write_manifest (import_xml_files excludes discovery)

## Evidence
`importable_xml` always skips tcp_discovery runs. When discovery succeeds (rc=0, valid XML, up hosts, open TCP) but both identify stages fail with no parseable host XML, `importable_xml()==[]` and `failed_runs()` is non-empty → status 'failed', exit 1, and stderr prints `error: 사용할 수 있는 스캔 결과가 없습니다(모든 단계 실패)`. Reproduced: `summary: live_hosts=1 ...` printed alongside the failure error and exit 1; `import_xml_files==[]` while `all_xml_files` still lists the discovery XML — an operator importing via the recommended list gets nothing despite real importable data on disk.

## Proposed fix
When `importable_xml()` is empty but a tcp_discovery run succeeded with non-empty live hosts/open ports, classify as 'partial' (exit 0) and include the discovery XML in `import_xml_files` as a fallback. Make the "모든 단계 실패" message conditional on discovery actually producing nothing.
