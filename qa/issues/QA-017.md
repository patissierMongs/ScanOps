# QA-017 — Non-ValueError/OSError exceptions (KeyError from malformed state file on --resume) escape main() as a raw traceback

- **group**: I    **disposition**: FIX    **severity**: medium    **dimension**: portability-edge
- **dedup_key**: `resume-malformed-state-keyerror-escapes-main`
- **location**: scanner/scanops_scanner.py:833-843 (main) and 586-598 (load_plan), 601-602 (print_plan)

## Evidence
```
main() catches only `except ValueError` and `except OSError`. load_plan (line 588) does `json.loads(...)` then only validates `plan.get('tool')`; it does not validate the schema. A valid-JSON state file that is missing required keys (output_dir, batches, cursor, base_flags, ...) passes load_plan and then KeyErrors deep inside execute/print_plan. Verified: a state file `{"tool":"scanops_scanner","nmap":"nmap"}` run with `--resume ... --dry-run` produces `KeyError: 'output_dir'` with a full traceback and exit code 1 (raw crash, not the clean `error: ...` path).
```
## Impact
A truncated/partially-written/older-format/hand-edited state.json gives the operator a Python traceback instead of a clean error message. In the GUI this dumps a stack trace into the run log. KeyError is the realistic case; any other non-ValueError/OSError raised inside execute() (e.g. a future code path) would similarly escape unfiltered.

## Proposed fix
Either validate the state-file shape in load_plan (check for required keys, raise ValueError with a clear message on missing keys) or broaden main()'s handler with a final `except Exception as exc:` that prints `error: ...` and returns a nonzero code, while keeping KeyboardInterrupt handling intact.
